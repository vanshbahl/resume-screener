# Recommendation Engine

## Overview

The Recommendation Engine translates intelligence and gaps into highly actionable insights for both candidates and recruiters. It sits at the top of the logic stack, completely isolated from parsing and raw heuristic matching.

## Architecture

We moved the `RecommendationService` into the `app/decision/` module and split it into discrete orchestrators:

- **`LearningRecommendationService`**: Triggers off missing critical skills inside a `GapAnalysis`. Maps directly to `config/recommendation/learning_paths.yaml` to suggest certifications or actionable study areas (e.g., "Build Backend Projects").
- **`CareerRecommendationService`**: Operates on a single `CandidateProfile`. Identifies heuristic signals (like `Leadership Potential`) generated by the `InsightService` to recommend next-level roles (e.g., Tech Lead) or specialization paths.
- **`HiringRecommendationService`**: Consumes `ScoreResult` and `RiskReport` to categorize candidates for recruiters: `Top Candidate`, `Hidden Gem`, `Backup`, or `High Risk`. Rules for this categorization live in `config/recommendation/recommendation_rules.yaml`.

## AI Extensibility

Currently, all recommendations are 100% deterministic, guaranteeing explainability without hallucinations.
The architecture is designed via interfaces (e.g., `BaseDecisionEngine`) so that an `LLMHiringRecommender` could be injected in the future, returning the exact same schema.
